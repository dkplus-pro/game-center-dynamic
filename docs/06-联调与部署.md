## 服务架构总览

经过 Wave 1-5 的搭建，项目现在包含四个独立服务：

```
┌──────────────────────────────────────────────────────────┐
│  浏览器访问                                                  │
│  ┌─────────────────────┐  ┌─────────────────────┐        │
│  │ 管理后台 (8080)      │  │ H5 端 (8081)         │        │
│  │ Modern.js + React   │  │ Modern.js + React   │        │
│  └────────┬────────────┘  └────────┬────────────┘        │
│           │ /api/*                  │ /api/*              │
│           ▼                         ▼                     │
│  ┌──────────────────────────────────────────────────┐    │
│  │  Go 后端 (3000)                                    │    │
│  │  Gin + GORM + SQLite                              │    │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │    │
│  │  │ handler  │→│ service  │→│ repo → SQLite DB │  │    │
│  │  └──────────┘ └──────────┘ └──────────────────┘  │    │
│  └──────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────┘
```

### 数据流

1. **管理后台** → 用户在编辑器拖拽组件 → 调用 `POST /api/pages` 保存 → 数据库存储
2. **管理后台** → 点击发布 → 调用 `POST /api/pages/:id/publish` → 状态变为 `published`
3. **H5 端** → 用户访问 `/:slug` → 调用 `GET /api/pages/:slug?resolve=true` → 获取页面 Schema + 游戏数据 → 渲染
4. **预览** → 管理后台点击预览 → 打开 H5 `/preview?pageId=xxx&draft=true` → 拉取草稿版本

## 启动所有服务

### 方式一：分别启动（推荐调试时使用）

需要打开 **3 个终端**：

```bash
# 终端 1：启动 Go 后端
cd server
go run cmd/server/main.go
# 输出：Server starting on :3000

# 终端 2：启动管理后台
cd apps/admin
pnpm dev
# 输出：Modern.js dev server running at http://localhost:8080

# 终端 3：启动 H5 端
cd apps/h5
pnpm dev
# 输出：Modern.js dev server running at http://localhost:8081
```

### 方式二：使用 Turbo 并行启动

```bash
# 在项目根目录一次性启动所有前端服务
pnpm dev
```

> **注意**：`pnpm dev` 只启动前端项目（admin + h5），Go 后端需要单独启动。

### 方式三：一键启动脚本

创建 `dev.sh` 脚本（项目根目录）：

```bash
#!/bin/bash
# 启动开发环境所有服务

echo "Starting Go backend..."
cd server && go run cmd/server/main.go &
BACKEND_PID=$!

echo "Starting admin (port 8080)..."
cd apps/admin && pnpm dev &
ADMIN_PID=$!

echo "Starting H5 (port 8081)..."
cd apps/h5 && pnpm dev &
H5_PID=$!

echo "All services started."
echo "  Backend: http://localhost:3000"
echo "  Admin:   http://localhost:8080"
echo "  H5:      http://localhost:8081"
echo ""
echo "Press Ctrl+C to stop all."

# Trap Ctrl+C to kill all services
trap "kill $BACKEND_PID $ADMIN_PID $H5_PID 2>/dev/null; exit" INT TERM
wait
```

### 前置条件

确保已安装以下工具：

| 工具 | 最低版本 | 验证命令 |
|------|---------|---------|
| Go | >= 1.21 | `go version` |
| Node.js | >= 18 | `node -v` |
| pnpm | >= 9.15 | `pnpm -v` |

执行步骤：

```bash
# 1. 安装依赖（首次运行）
pnpm install

# 2. 启动后端
cd server && go run cmd/server/main.go

# 3. 启动前端（新终端）
cd apps/admin && pnpm dev   # 终端 2
cd apps/h5 && pnpm dev      # 终端 3
```

## 端口分配与 API 代理

### 端口规划

| 服务 | 端口 | 说明 |
|------|------|------|
| Go 后端 | 3000 | Gin HTTP 服务器，提供 REST API |
| 管理后台 | 8080 | Modern.js 开发服务器，含代理 |
| H5 端 | 8081 | Modern.js 开发服务器，含代理 |

### API 代理配置

前端应用通过 Modern.js 的 dev proxy 将 `/api/*` 请求转发到后端：

**apps/admin/modern.config.ts**：
```typescript
export default defineConfig({
  server: { port: 8080 },
  dev: {
    proxy: { '/api': 'http://localhost:3000' },
  },
});
```

**apps/h5/modern.config.ts**：
```typescript
export default defineConfig({
  server: { port: 8081 },
  dev: {
    proxy: { '/api': 'http://localhost:3000' },
  },
});
```

### CORS 配置

后端通过 `gin-contrib/cors` 中间件配置跨域：

```go
r.Use(cors.New(cors.Config{
    AllowOrigins:     []string{"http://localhost:8080", "http://localhost:8081"},
    AllowMethods:     []string{"GET", "POST", "PUT", "DELETE", "OPTIONS"},
    AllowHeaders:     []string{"Origin", "Content-Type", "Authorization"},
    AllowCredentials: true,
}))
```

### 请求链路

```
浏览器 → http://localhost:8080/api/pages
         ↓
Modern.js dev proxy → http://localhost:3000/api/pages
         ↓
Gin router → handler.ListPages → service → repo → SQLite
```

## 端到端联调流程

以下是完整的端到端验证流程，确保前后端联调正常：

### 步骤 1：验证后端 API

```bash
# 启动后端后，在另一个终端验证 API
curl http://localhost:3000/api/pages
# 应该返回：{"data":[]}

# 创建页面
curl -X POST http://localhost:3000/api/pages \
  -H "Content-Type: application/json" \
  -d '{"name":"测试页面","slug":"test-page"}'
# 返回：{"data":{"id":"...","slug":"test-page"}}

# 获取页面
curl "http://localhost:3000/api/pages/test-page?resolve=true"
# 返回页面 Schema JSON

# 获取游戏列表
curl http://localhost:3000/api/games
# 返回：{"data":[]}
```

### 步骤 2：验证管理后台

```
1. 打开浏览器访问 http://localhost:8080
2. 应该看到页面列表（空状态："暂无页面，点击新建开始搭建"）
3. 点击 "+ 新建页面" 按钮
4. 浏览器跳转到 /editor/{slug} 编辑器页面
5. 编辑器显示三栏布局：
   - 左侧：组件面板（"组件面板"标题 + 组件列表）
   - 中间：预览画布（"从左侧拖入组件开始搭建"）
   - 右侧：属性面板（"点击组件编辑属性"）
6. 从左侧拖入 "横幅轮播" 组件到画布
7. 点击画布上的组件，右侧属性面板显示其配置
8. 点击 "保存" 按钮，数据持久化到后端
```

### 步骤 3：验证 H5 端

```
1. 打开浏览器访问 http://localhost:8081/test-page
   （替换为步骤 2 创建的页面 slug）
2. 看到页面渲染，包含顶部标题和组件内容
3. 访问不存在的页面 http://localhost:8081/non-existent
4. 看到友好的错误页面："页面加载失败" + "重试" 按钮
```

### 步骤 4：验证预览功能

```
1. 在管理后台编辑器中，点击 "预览" 按钮
2. 新标签页打开 http://localhost:8081/preview?pageId=xxx&draft=true
3. 页面顶部显示黄色 "预览模式" 横幅
4. 页面内容与编辑器中的一致
```

```bash
# 方式 1：在项目根目录运行所有 E2E 测试
pnpm e2e

pnpm e2e

# 方式 3：只运行 H5 端的 E2E 测试
cd apps/h5
pnpm e2e

# 方式 4：使用 Playwright UI 模式（可视化调试）
cd apps/admin
pnpm e2e:ui
```

### 测试策略说明

- **Smoke Test（冒烟测试）**：只验证核心 UI 元素存在、无控制台报错
- **不依赖真实组件数据**：测试使用空状态或 404 场景，不依赖预置数据
- **控制台错误检查**：每个测试收集 `console.error`，排除 `favicon` 等无关错误
- **独立运行**：每个测试可独立运行，不依赖共享状态

### 添加新测试用例

参考现有测试结构，添加新的测试用例：

```typescript
test('your test name', async ({ page }) => {
  // 1. 设置错误收集器
  const errors: string[] = [];
  page.on('console', (msg) => {
    if (msg.type() === 'error') errors.push(msg.text());
  });

  // 2. 导航到页面
  await page.goto('/your-route');

  // 3. 断言关键元素
  await expect(page.locator('text=关键文本')).toBeVisible();

  // 4. 验证无控制台错误
  expect(errors.filter(e => !e.includes('favicon'))).toHaveLength(0);
});
```

## 常见问题排查

### Q: 后端启动失败 `Error: port 3000 already in use`

```bash
# 查找占用 3000 端口的进程
lsof -i :3000
# 杀掉进程
kill -9 <PID>
# 或换一个端口：修改 server/cmd/server/main.go 中的 :3000
```

### Q: 前端启动后提示 `Cannot GET /api/pages`

**原因**：后端未启动或代理未生效。

**解决**：
1. 确认后端已启动：`curl http://localhost:3000/api/pages`
2. 确认 Modern.js 代理配置正确（`modern.config.ts` 中的 `dev.proxy`）
3. 重启前端开发服务器：`pnpm dev`

### Q: 管理后台报 CORS 错误

**原因**：后端 CORS 配置未包含当前前端端口。

**解决**：检查 `server/cmd/server/main.go` 中的 `AllowOrigins` 配置，确保包含 `http://localhost:8080` 和 `http://localhost:8081`。

### Q: `pnpm install` 报错 `Unsupported engine`

```bash
# 检查 Node.js 版本
node -v  # 需要 >= 18

# 如果版本过低，使用 nvm 切换
nvm install 18
nvm use 18
```

### Q: Playwright 测试报 `browserType.launch: Executable doesn't exist`

```bash
# 安装 Playwright 浏览器
cd apps/admin
npx playwright install chromium

cd apps/h5
npx playwright install chromium
```

### Q: 组件库构建失败 `tsup: command not found`

```bash
# 确保已安装所有依赖
pnpm install

# 单独构建组件库
cd packages/components
pnpm build
```

### Q: 数据库文件在哪里？

SQLite 数据库文件位于 `server/data/game_center.db`。每次启动后端时自动创建。

```bash
# 查看数据库内容
sqlite3 server/data/game_center.db ".tables"
sqlite3 server/data/game_center.db "SELECT * FROM pages;"
```

### Q: 如何重置数据库？

```bash
rm server/data/game_center.db
# 重启后端，自动创建空数据库
```
