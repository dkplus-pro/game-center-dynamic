# 05 - H5 前端搭建

## 概述

`apps/h5` 是 game-center-dynamic 面向终端用户的移动端 H5 应用。基于 Modern.js v2 框架，采用文件路由、Tailwind CSS 移动端优先设计，通过 SchemaRenderer 引擎将后端页面配置动态渲染为真实 UI。

## 架构设计

### 技术栈

| 技术 | 用途 |
|------|------|
| Modern.js v2 | 应用框架，提供文件路由、构建、开发服务器 |
| Tailwind CSS | 原子化 CSS 工具库，移动端优先 |
| React 18 | UI 渲染层 |
| pnpm workspace | monorepo 包管理 |

```
apps/h5/src/
├── modern-app-env.d.ts     # Modern.js 类型声明
├── routes/
│   ├── layout.tsx          # 全局布局（全屏无头）

```

## 核心设计

### 1. SchemaRenderer 引擎

SchemaRenderer 是本应用的核心——它将后端返回的页面 schema（组件列表）解析为真实的 React 组件树。

**工作流程：**

```
后端 API (resolve=true)
    │
    ▼
usePageData(slug)
    │ 返回 PageSchema { components: PageComponent[] }
    ▼
SchemaRenderer
    ├── 1. 按 order 排序组件
    ├── 2. 解析每个组件的 propsJson
    ├── 3. 从 componentRegistry 查找组件实现
    ├── 4. 未知类型 → 跳过 + 控制台警告
    └── 5. 包裹 ErrorBoundary + DataProvider 后渲染
```

**关键代码：**

```typescript
const entry = componentRegistry[comp.type];
if (!entry) {
  console.warn(`[SchemaRenderer] Unknown component type: "${comp.type}"`);
  return null;
}

const { resolvedData, ...restProps } = JSON.parse(comp.propsJson);

return (
  <ErrorBoundary key={comp.id} fallback={<ComponentErrorFallback type={comp.type} />}>
    <DataProvider resolvedData={resolvedData}>
      {(data) => {
        const Comp = entry.component;
        return <Comp {...restProps} data={data} />;
      }}
    </DataProvider>
  </ErrorBoundary>
);
```

### 2. DataProvider + resolve 模式

**设计决策：服务端数据解析**

传统做法是客户端分别调用游戏数据 API，但这会导致：
- 多次网络请求，增加加载时间
- 组件间数据依赖管理复杂
- 首屏渲染慢

本项目的方案是：**后端在返回页面 schema 时，通过 `?resolve=true` 参数预先解析所有组件的游戏数据，嵌入到 `resolvedData` 字段中。**

```
GET /api/pages/:slug?resolve=true
→ 返回 PageSchema，每个组件的 propsJson 中包含 resolvedData: GameInfo[]
```

DataProvider 因此简化为一个透传层：

```typescript
export function DataProvider({ resolvedData, children }) {
  return <>{children(resolvedData)}</>;
}
```

当 `resolvedData` 为 `undefined` 时（API 未使用 resolve），组件内部自行处理空状态（如 GameGrid2x4 的骨架屏）。

### 3. 三层降级策略

H5 应用实现了三层错误/降级处理，确保任何时候都有合理的用户体验：

| 层级 | 组件 | 处理范围 | 降级表现 |
|------|------|----------|----------|
| Layer 3 | PageLoading / PageError | 页面整体获取失败 | 骨架屏 → 错误提示 + 重试按钮 |
| Layer 2 | DataProvider | 组件数据为空 | 透传 undefined，组件内部处理空状态 |
| Layer 1 | ErrorBoundary | 单个组件渲染异常 | 红色提示 "组件渲染失败，已跳过" |

**Layer 1 示例：**

```typescript
export class ErrorBoundary extends React.Component {
  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return (
        <div className="border border-red-200 bg-red-50 p-4 text-sm text-red-600">
          ⚠️ 组件渲染失败，已跳过
        </div>
      );
    }
    return this.props.children;
  }
}
```

**Layer 3 示例：**

```typescript
function PageError({ error }: { error: string | null }) {
  return (
    <div className="flex flex-col items-center justify-center">
      <div className="text-4xl mb-4">😞</div>
      <h2>页面加载失败</h2>
      <p>{error || '请稍后再试'}</p>
      <button onClick={() => window.location.reload()}>重试</button>
    </div>
  );
}
```

### 4. 预览模式

预览模式通过 URL 参数 `?draft=true` 触发，用于管理后台实时预览未发布的页面。

**发布态（end user）：**
```
/h5/:slug
→ usePageData(slug, false)
→ GET /api/pages/:slug?resolve=true
```

**预览态（admin）：**
```
/h5/preview?pageId=xxx&draft=true
→ usePageData(pageId, true)
→ GET /api/pages/xxx?resolve=true&draft=true
```

预览页面顶部会渲染一个黄色提示条 "🔍 预览模式"，与发布态做视觉区分。

### 5. 移动端优先设计

- **最大宽度 480px**（`max-w-md`），超宽屏幕居中显示
- **Tailwind CSS 原子类**，无自定义 CSS（除全局重置）
- **系统字体栈**：`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- **粘性标题**：页面标题使用 `sticky top-0` + `backdrop-blur` 毛玻璃效果
- **骨架屏**：使用 `animate-pulse` 实现加载占位

## API 约定

### 页面数据获取

```
GET /api/pages/{slug}?resolve=true
```

**响应：** `PageSchema`（来自 `@game-center/types/generated/common`）

```typescript
interface PageSchema {
  id: string;
  name: string;
  slug: string;
  version: number;
  status: string;          // "published" | "draft"
  components: PageComponent[];
  createdAt: number;
  updatedAt: number;
}

interface PageComponent {
  id: string;
  type: string;           // "Banner" | "GameGrid2x4" | ...
  order: number;
  propsJson: string;       // JSON string, 解析后包含 resolvedData?: GameInfo[]
}
```

### 开发代理

Modern.js 开发服务器在 8081 端口，`/api` 请求代理到后端 3000 端口：

```typescript
// modern.config.ts
dev: {
  proxy: {
    '/api': 'http://localhost:3000',
  },
},
```

## 组件注册表

H5 应用通过 `@game-center/components` 的 `componentRegistry` 获取组件实现：

| 类型 | 组件 | 说明 |
|------|------|------|
| `Banner` | `<Banner images autoplay interval />` | 横幅轮播 |
| `GameGrid2x4` | `<GameGrid2x4 title data />` | 2×N 游戏网格 |
| `GameCardSwiper` | `<GameCardSwiper title data />` | 横向滑动卡片 |
| `GameGrid4x1` | `<GameGrid4x1 data />` | 纵向列表布局 |
| `GameList` | `<GameList title data />` | 分页列表 |

所有组件渲染时接收 `data` prop（`GameInfo[]`），数据来自后端 `resolve` 阶段。
