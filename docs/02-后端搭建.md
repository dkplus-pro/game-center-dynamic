# 02 - 后端搭建

## 概述

本节我们将搭建游戏中心项目的 Go 后端服务，包含数据库设计、API 接口、分层架构和代码生成工具。

## 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 语言 | Go 1.21+ | 高性能、简洁 |
| Web 框架 | Gin | 轻量级 HTTP 框架 |
| ORM | GORM | 强大的 Go ORM |

| IDL | Thrift | 接口定义语言 |

## Thrift IDL 设计

Thrift IDL（接口定义语言）用于描述数据结构和 API 接口，方便前后端类型一致，避免手动同步类型定义。

### 目录结构

```
server/idl/
├── common.thrift        # 公共数据结构
├── page_service.thrift  # 页面服务接口
└── game_service.thrift  # 游戏服务接口
```

### 公共数据结构 (`common.thrift`)

定义了三个核心数据结构，是所有服务的类型基础：

- **GameInfo**：游戏信息，包含 ID、名称、图标、描述、标签（多选）、评分、分类
- **PageComponent**：页面组件，描述单个 UI 组件的类型、排序和配置信息
- **PageSchema**：页面 Schema，包含名称、Slug、版本号、状态和组件列表

```thrift
struct GameInfo {
  1: string id,
  2: string name,
  3: string icon,
  4: string description,
  5: list<string> tags,
  6: double rating,
  7: string category,
}

struct PageComponent {
  1: string id,
  2: string type,
  3: i32 order,
  4: string propsJson,
}

struct PageSchema {
  1: string id,
  2: string name,
  3: string slug,
  4: i32 version,
  5: string status,
  6: list<PageComponent> components,
  7: i64 createdAt,
  8: i64 updatedAt,
}
```

## 分层架构

项目采用经典的 **Handler → Service → Repository → Model** 分层架构，职责清晰，便于测试和维护：

```
请求 → Handler (HTTP 参数解析) → Service (业务逻辑) → Repository (数据访问) → Model (GORM)
```

### Model 层（数据模型）

使用 GORM 标签定义数据库表结构。这里的设计要点：

- 使用字符串 UUID 作为主键，避免自增 ID 的可预测性
- `Tags` 字段以 JSON 数组字符串存储（`["RPG","卡牌"]`），避免额外关联表
- `PageComponent` 存储组件树的完整 JSON，支持版本管理
- 使用毫秒级时间戳（`int64`）而非 `time.Time`，与前端 JavaScript 兼容

```go
// model/page.go
type Page struct {
    ID        string `gorm:"primaryKey;size:36" json:"id"`
    Name      string `gorm:"size:100;not null" json:"name"`
    Slug      string `gorm:"size:100;uniqueIndex;not null" json:"slug"`
    Status    string `gorm:"size:20;default:draft" json:"status"`
    Version   int    `gorm:"default:1" json:"version"`
    CreatedAt int64  `json:"createdAt"`
    UpdatedAt int64  `json:"updatedAt"`
}

type PageComponent struct {
    ID         string `gorm:"primaryKey;size:36" json:"id"`
    PageID     string `gorm:"size:36;index;not null" json:"pageId"`
    Version    int    `gorm:"not null" json:"version"`
    Components string `gorm:"type:text" json:"components"`
    CreatedAt  int64  `json:"createdAt"`
}

// model/game.go
type Game struct {
    ID          string  `gorm:"primaryKey;size:36" json:"id"`
    Name        string  `gorm:"size:100;not null" json:"name"`
    Icon        string  `gorm:"size:500" json:"icon"`
    Description string  `gorm:"type:text" json:"description"`
    Tags        string  `gorm:"type:text" json:"tags"`
    Rating      float64 `json:"rating"`
    Category    string  `gorm:"size:50" json:"category"`
    CreatedAt   int64   `json:"createdAt"`
    UpdatedAt   int64   `json:"updatedAt"`
}
```

### Repository 层（数据访问）

封装所有数据库操作，提供清晰的数据访问接口。关键实现：

- `GameRepo.List()` 支持多标签 OR 查询（匹配任一标签即返回）
- `GameRepo.ListTags()` 从已有游戏数据中提取并去重标签
- `PageRepo.SoftDelete()` 通过设置 `status=archived` 实现软删除

```go
// repo/page_repo.go
type PageRepo struct {
    db *gorm.DB
}

func (r *PageRepo) List() ([]model.Page, error) {
    var pages []model.Page
    err := r.db.Where("status != ?", "archived").Order("created_at DESC").Find(&pages).Error
    return pages, err
}

func (r *PageRepo) GetBySlug(slug string) (*model.Page, error) {
    var page model.Page
    err := r.db.Where("slug = ? AND status != ?", slug, "archived").First(&page).Error
    return &page, err
}
```

### Service 层（业务逻辑）

核心业务逻辑，包含两个重要特性：

#### 1. 页面数据解析（resolve）

当 `GetPage` 接口传入 `resolve=true` 时，自动解析页面组件中的 `dataSource` 配置：

- **filter 类型**：`{"type":"filter","tags":"RPG","sortBy":"rating","limit":10}` → 自动查询符合条件的游戏
- **manual 类型**：`{"type":"manual","ids":["id1","id2"]}` → 按 ID 批量获取游戏
- 解析后的游戏数据注入到组件的 `resolvedData` 字段，前端无需额外请求

```go
func (s *PageService) resolveComponents(components []map[string]interface{}) ([]map[string]interface{}, error) {
    for i, comp := range components {
        props := comp["props"].(map[string]interface{})
        dataSource := props["dataSource"].(map[string]interface{})
        
        switch dataSource["type"] {
        case "filter":
            games, _ := s.gameRepo.List(tags, sortBy, limit)
            components[i]["resolvedData"] = gameListToMaps(games)
        case "manual":
            games, _ := s.gameRepo.BatchGet(ids)
            components[i]["resolvedData"] = gameListToMaps(games)
        }
    }
    return components, nil
}
```

#### 2. 版本管理

页面更新时自动递增版本号，组件变更创建新版本记录。

### Handler 层（HTTP 接口）

基于 Gin 框架的 RESTful API 实现，所有接口统一返回 JSON 格式：

```json
{ "data": ... }
{ "error": "错误描述" }
```

**完整 API 列表：**

| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| GET | `/api/pages` | 页面列表 | - |
| GET | `/api/pages/:slug` | 页面详情 | `?resolve=true` 解析数据源 |
| POST | `/api/pages` | 创建页面 | `{"name":"","slug":""}` |
| PUT | `/api/pages/:id` | 更新页面 | 完整 Page JSON |
| POST | `/api/pages/:id/publish` | 发布页面 | - |
| DELETE | `/api/pages/:id` | 删除页面（软删除） | - |
| GET | `/api/games` | 游戏列表 | `?tags=RPG,卡牌&sortBy=rating&limit=10` |
| GET | `/api/games/batch` | 批量获取游戏 | `?ids=id1,id2,id3` |
| GET | `/api/game-tags` | 所有标签 | - |

## 代码生成工具

`cmd/codegen/main.go` 是一个轻量级 Thrift IDL 解析器，自动将 `.thrift` 文件转换为 TypeScript 类型定义，确保前后端类型完全一致。

### 使用方法

```bash
cd server
go run cmd/codegen/main.go
```

### 生成结果

生成文件位于 `packages/types/src/generated/`：

**common.ts**：
```typescript
// Auto-generated from Thrift IDL — do not edit manually.

export interface GameInfo {
  id: string;
  name: string;
  icon: string;
  description: string;
  tags: string[];
  rating: number;
  category: string;
}

export interface PageComponent {
  id: string;
  type: string;
  order: number;
  propsJson: string;
}

export interface PageSchema {
  id: string;
  name: string;
  slug: string;
  version: number;
  status: string;
  components: PageComponent[];
  createdAt: number;
  updatedAt: number;
}
```

### 实现原理

- 使用正则表达式解析 Thrift struct 定义（`struct Name { ... }`）
- 解析字段定义（`序号: 类型 字段名`）和 `list<T>` 泛型
- 类型映射：`i32/i64/double` → `number`，`string` → `string`，`bool` → `boolean`
- 输出标准 TypeScript 接口定义文件

## CORS 配置

允许以下来源的跨域请求（前端开发服务器地址）：

- `http://localhost:8080`
- `http://localhost:8081`

支持的方法：`GET, POST, PUT, DELETE, OPTIONS`，允许携带 `Authorization` 请求头。

## 启动服务

```bash
cd server
go run cmd/server/main.go
```

服务运行在 `http://localhost:3000`，启动后自动创建 `data/game_center.db` 并填充种子数据。
