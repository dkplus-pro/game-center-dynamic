# 管理后台搭建

## 概述

管理后台（`apps/admin`）是一个基于 **Modern.js v2** 的页面搭建工具，为运营人员提供可视化的页面编辑能力。支持拖拽式组件编排、实时预览、属性编辑，以及草稿保存/发布工作流。

## 技术栈

| 技术 | 用途 |
|------|------|
| Modern.js v2 | 应用框架，内置路由、构建、开发服务器 |
| React 18 | UI 渲染 |
| TypeScript | 类型安全 |
| Tailwind CSS | 原子化样式 |
| Zustand v5 | 状态管理 |
| @dnd-kit | 拖拽排序 |

## Zustand 状态管理

### 数据结构

```typescript
interface EditorState {
  pageId: string;          // 当前页面 ID
  pageName: string;        // 页面名称
  slug: string;            // URL 标识
  status: 'draft' | 'published';

  components: ComponentInstance[];  // 画布上的组件列表
  selectedId: string | null;        // 当前选中的组件

  isDirty: boolean;                 // 是否有未保存修改
  lastSavedSnapshot: string;        // 最后保存时的序列化快照

  history: ComponentInstance[][];   // 撤销历史栈
  historyIndex: number;             // 当前历史位置

  // Actions
  addComponent(type, index?): void;
  removeComponent(id): void;
  moveComponent(fromIndex, toIndex): void;
  updateComponentProps(id, props): void;
  selectComponent(id | null): void;
  undo(): void;
  redo(): void;
  saveDraft(): Promise<void>;
  publish(): Promise<void>;
  loadPage(slug): Promise<void>;
}
```

### 撤销/重做

- 每次组件变更（添加、删除、移动、属性修改）都会在历史栈中 push 一个快照
- 历史栈上限 50 条，超出时自动移除最旧条目
- `undo()` 将 `historyIndex` 减 1 并恢复对应快照
- `redo()` 将 `historyIndex` 加 1 并恢复对应快照
- 任何新操作会截断 redo 历史（无法重做已覆盖的操作）

### 属性修改防抖

`updateComponentProps` 采用 300ms 防抖策略：
1. 多次连续调用先合并到 `pendingProps` 缓冲区
2. 乐观更新 UI（立即反映在画布上）
3. 300ms 后统一推入历史栈

### 脏状态追踪

- 每次修改后，将当前组件列表序列化并与 `lastSavedSnapshot` 比较
- 保存成功后更新 `lastSavedSnapshot`，`isDirty` 重置为 false
- TopBar 中显示橙色圆点指示未保存状态

## 拖拽系统 (@dnd-kit)

### 组件面板 → 画布

1. 组件面板中的每个组件使用 `useDraggable` 包裹，`data.source = 'panel'`
2. 拖拽到画布时，`DragEndEvent` 中检测 `active.data.current.source === 'panel'`
3. 从 `active.data.current.type` 获取组件类型
4. 调用 `store.addComponent(type, dropIndex)` 插入组件

### 画布内排序

1. 画布中的组件使用 `useSortable` 包裹
2. 通过 `SortableContext` 提供排序策略
3. 拖拽结束时，检测 `active.id !== over.id`
4. 调用 `store.moveComponent(oldIndex, newIndex)` 重新排序

### 拖拽覆盖层

`DragOverlay` 在拖拽过程中显示一个浮动卡片，展示正在拖拽的组件名称和图标。

## 组件注册表集成

管理后台通过 `@game-center/components` 包获取组件信息：

```typescript
import { componentList, componentRegistry } from '@game-center/components';

// componentList: [{ type, label, icon }] — 用于组件面板
// componentRegistry: { [type]: { component, schema, PropsEditor } } — 用于渲染和属性编辑
```

### 画布渲染

```typescript
const entry = componentRegistry[comp.type];
const Comp = entry.component;
<Comp {...comp.props} data={[]} />  // 管理后台预览无游戏数据，传空数组
```

### 属性编辑

1. 选中组件时，PropsPanel 从 `componentRegistry[comp.type]` 获取 `PropsEditor` 和 `schema`
2. 使用 `Suspense` 包裹懒加载的 `PropsEditor`
3. 同时渲染 `PropsEditorFactory` 作为通用属性编辑器（基于 `schema.propsMeta`）

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/pages` | 获取页面列表 |
| GET | `/api/pages/:slug?resolve=true` | 获取页面详情（含组件） |
| POST | `/api/pages` | 创建新页面 |
| PUT | `/api/pages/:id` | 更新页面（名称、组件） |
| POST | `/api/pages/:id/publish` | 发布页面 |

开发环境下，`/api` 请求通过 Modern.js 代理到 `http://localhost:3000`。

## 保存/发布工作流

```
编辑组件 → 修改属性 → 脏状态标记 → 保存草稿（PUT）
                                    ↓
                               发布（POST /publish）
                                    ↓
                               status = 'published'
```

1. **保存草稿**：调用 `PUT /api/pages/:id`，提交当前页面名称和组件列表
2. **发布**：调用 `POST /api/pages/:id/publish`，后端将状态改为 `published`
3. **预览**：新标签页打开 `/h5/preview?slug=xxx&draft=true`

## 配置说明

- **端口**：8080（`modern.config.ts` 中 `server.port`）
- **API 代理**：`/api` → `http://localhost:3000`（`dev.proxy`）
- **Tailwind 扫描**：同时扫描 `apps/admin/src` 和 `packages/components/src`，确保组件库的 Tailwind 类在生产构建中生效
- **路由**：Modern.js 文件系统路由，`src/routes/` 下的文件自动映射为 URL 路径
