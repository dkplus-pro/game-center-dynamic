# 03 - 组件库搭建

## 概述

`@game-center/components` 是 game-center-dynamic 项目共享组件库，为管理后台（admin）和移动端 H5（h5）提供统一的游戏中心展示组件。

## 架构设计

### 双重目的：渲染 + 编辑

每个组件同时服务于两个场景：

| 场景 | 使用者 | 用途 |
|------|--------|------|
| 渲染端 | H5 / 预览 | 将组件 props 渲染为真实 UI |
| 编辑端 | Admin | 通过表单编辑组件属性，实时预览效果 |

为实现这个双重目的，每个组件拆分为三个文件：

```
src/Banner/
├── index.tsx       # 渲染组件（纯展示，接收 resolved data）
├── schema.ts       # 组件元数据（类型、标签、默认属性、属性元数据）
└── PropsEditor.tsx  # 管理端属性编辑表单（委托给 PropsEditorFactory）
```

### 组件注册表模式

`registry.ts` 将组件类型字符串映射到实际实现，实现动态组件加载：

```typescript
const entry = componentRegistry['Banner'];
// entry.component   → React 组件（同步导入，首屏需要）
// entry.schema      → 元数据（同步导入，用于表单生成）
// entry.PropsEditor → 管理表单（lazy 导入，按需加载）
```

**设计决策：**
- `component` 和 `schema` 同步导入 — 渲染时必须立即可用
- `PropsEditor` 使用 `React.lazy()` — 管理端才需要，减少首屏体积

## 组件详解

### 1. Banner（横幅轮播）

**功能：** 自动轮播图片，支持手动切换。

**核心实现：**
- 使用 CSS `transform: translateX()` 实现平滑滑动
- `useEffect` + `setInterval` 管理自动播放定时器
- 上一张/下一张按钮 + 底部指示点导航
- 空状态：显示 "暂无图片" 占位符

**属性：**
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| images | string[] | [] | 图片 URL 数组 |
| autoplay | boolean | true | 是否自动播放 |
| interval | number | 3000 | 自动播放间隔(ms) |
| height | number | 200 | 轮播图高度(px) |

### 2. GameCard（游戏卡片 — 共享子组件）

**功能：** 可复用的游戏信息卡片，被所有游戏列表组件使用。

**核心实现：**
- 图标 48×48，无图标时显示 🎮 占位符
- 排名圆形徽章（蓝色背景，仅 showRank=true 时显示）
- 星级评分（★ 字符，黄色/灰色）
- 标签徽章（蓝色背景小标签）
- 可选描述文本（`line-clamp-2` 截断）
- 支持 `onClick` 事件 + 键盘无障碍（Enter 键触发）

### 3. GameGrid2x4（2行×N列网格）

**功能：** 游戏卡片网格布局，CSS Grid 实现。

**核心实现：**
- 动态列数（通过 `gridTemplateColumns` style 控制）
- 最多显示 `columns × 2` 个卡片
- 三种状态：骨架屏加载 → 空数据 → 正常渲染

### 4. GameCardSwiper（卡片轮播）

**功能：** 横向可滚动的卡片行，带 snap 效果。

**核心实现：**
- CSS `scroll-snap-type: x mandatory` 实现对齐吸附
- 每张卡片 `flex-shrink: 0` + 动态宽度计算
- 左右箭头按钮，到达边界时自动隐藏
- `onScroll` 事件检测可滚动状态

### 5. GameGrid4x1（4行×1列列表）

**功能：** 纵向列表，每个卡片占满一行。

**核心实现：**
- 使用 `divide-y` 分隔线分隔卡片
- 最多显示 4 个游戏
- 可选显示描述文本

### 6. GameList（支持加载更多的游戏列表）

**功能：** 纵向列表，支持分页加载。

**核心实现：**
- 内部维护 `visibleCount` 状态管理分页
- 每次点击 "加载更多" 增加 `pageSize` 项
- 全部加载完成后自动隐藏按钮

## PropsEditorFactory 通用表单引擎

`PropsEditorFactory` 是管理端的核心组件，根据 `propsMeta` 自动生成表单控件。

**支持的控件类型：**

| 类型 | 控件 | 说明 |
|------|------|------|
| text | `<input type="text">` | 文本输入，支持 maxLength |
| boolean | `<input type="checkbox">` | 开关切换，显示启用/关闭状态 |
| select | `<select>` | 下拉选择，options 配置 |
| color | `<input type="color">` | 颜色选择器，实时预览色值 |
| image-list | 文本输入 + 缩略图预览 | 逗号分隔 URL，支持 maxCount |
| game-data-source | 类型选择 + 条件筛选 | 规则圈选 / 手动选择 |
| number | `<input type="number">` | 数字输入，支持 min/max/step |

**设计决策：**
- 使用 discriminated union（`PropMeta`）保证类型安全
- 每个控件统一 `handleChange` 回调，不修改原对象
- `values` 来自外部状态（受控组件模式）

## 类型系统

```typescript
// 游戏信息（后端返回的标准结构）
GameInfo { id, name, icon, description, tags, rating, category }

// 数据源配置（管理端配置，非运行时数据）
DataSource { type: 'filter' | 'manual', filters?, gameIds? }

// 组件元数据
ComponentSchema { type, label, icon, defaultProps, propsMeta }

// 属性元数据（discriminated union）
PropMeta = TextMeta | BooleanMeta | SelectMeta | ColorMeta | ImageListMeta | GameDataSourceMeta | NumberMeta
```

**重要原则：** 组件接收的是已解析的 `data: GameInfo[]`，而不是 `dataSource`。数据解析逻辑由页面层或服务端负责。

## 如何添加新组件

1. 在 `src/` 下创建组件目录，如 `src/NewComponent/`
2. 创建 `index.tsx`（渲染组件）
3. 创建 `schema.ts`（元数据配置）
4. 创建 `PropsEditor.tsx`（包装 PropsEditorFactory）
5. 在 `registry.ts` 中注册
6. 在 `index.ts` 中导出
7. 编写 `__tests__/index.test.tsx`（至少 3 个测试用例）

### 示例：注册新组件

```typescript
// registry.ts
NewComponent: {
  component: NewComponent,
  schema: NewComponentSchema,
  PropsEditor: lazy(() =>
    import('./NewComponent/PropsEditor').then(m => ({ default: m.NewComponentPropsEditor }))
  ),
}
```

## 测试策略

使用 Vitest + @testing-library/react 进行单元测试。

**测试覆盖要点：**
- ✅ 正常渲染：传入有效 props，验证 DOM 输出
- ✅ 空状态：空数组 / undefined 数据时的占位符
- ✅ 交互行为：点击、输入等用户操作
- ✅ 边界条件：空图标、0 数据、边界索引
- ✅ 加载状态：骨架屏渲染

运行测试：
```bash
cd packages/components
pnpm test          # 单次运行
pnpm test:watch    # 监听模式
```

## 构建配置

使用 tsup 打包：
- 入口：`src/index.ts`
- 输出：CJS + ESM 双格式 + 类型声明
- 外部依赖：react、react-dom（不打包进 bundle）

```bash
pnpm build    # 构建输出到 dist/
```

## 样式方案

所有组件使用 Tailwind CSS 工具类，不依赖任何第三方组件库。

- 响应式布局通过 Tailwind 响应式前缀（`sm:`、`md:` 等）
- 应用端通过 `tailwind.config.ts` 的 `content` 扫描组件源码
- 无需 CSS Modules，保持零运行时开销
